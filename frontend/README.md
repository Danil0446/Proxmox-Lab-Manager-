## Панель администратора для стендов студентов

Фронтенд-приложение для системного администратора, который заходит на сайт и поднимает стенды для студентов. Реализовано **только на фронтенде** (React + Vite, Node.js используется как окружение для сборки и dev-сервера).

### Стек

- **Node.js** (окружение для запуска инструментов)
- **Vite** — сборка и dev-сервер
- **React 18 + TypeScript** — UI

### Запуск проекта

1. Установите зависимости:

```bash
npm install
```

2. Запустите dev-сервер:

```bash
npm run dev
```

3. Откройте в браузере адрес, который покажет Vite (обычно `http://localhost:5173/`).

### Что уже реализовано во фронтенде

- **Авторизация администратора** — доступ к панели только после входа:
  - тестовый логин: `admin`
  - тестовый пароль: `admin123`
- **Список студентов** с полями: ФИО, группа, email, статус стенда.
- **Поиск** по имени, группе или email.
- **Множественный выбор студентов** (чекбоксы, кнопка "Выделить всех на странице").
- Кнопка **«Поднять стенды (N)»**:
  - при нажатии меняет статус выбранных студентов на `Запуск...`,
  - через 1.5 секунды меняет статус на `Работает` (имитация ответа backend).
- Кнопка **«Выключить стенды»** для отключения стендов у выбранных студентов.
- Отображение **времени работы** стенда и **ресурсов** (CPU, RAM, диск).
- Красивный тёмный интерфейс, адаптивная верстка.

Логика находится в компоненте `App` (`src/App.tsx`), а отображение списка — в `src/components/StudentList.tsx`.

### Интеграция с реальным backend

Сейчас логика запуска стендов **эмулируется** через `setTimeout` внутри функции `startStandsForSelected` в `src/App.tsx`.

Чтобы подключить реальный backend:

1. Замените внутри `startStandsForSelected` блок с `setTimeout` на вызов вашего API, например:

```ts
await fetch("https://your-api/students/stands/start", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ studentIds: selectedIds })
});
```

2. В обработке ответа обновляйте статус студентов (`standStatus`) в соответствии с тем, что вернёт API (успех/ошибка).

3. При желании можно:
   - добавить кнопки **Остановить стенды**, **Перезапустить**, **Пересоздать**;
   - добавить фильтр по статусу стенда;
   - подключить авторизацию администратора.

### Структура проекта (основное)

- `index.html` — корневой HTML.
- `src/main.tsx` — вход в React-приложение.
- `src/App.tsx` — основная логика админ-панели (список, поиск, запуск стендов).
- `src/components/StudentList.tsx` — таблица со студентами.
- `src/components/StandActionButton.tsx` — общая кнопка действия.
- `src/components/StandStatusBadge.tsx` — бейджи статусов стенда.
- `src/styles.css` — стили (тёмная тема, таблица, кнопки и т.д.).
